# Credible — AI Learning Validation & Certification Platform

Credible is an open-source Learning Validation Platform that transforms passive video consumption into verifiable proof of knowledge. By integrating with an external AI pipeline (such as an n8n video analysis workflow), Credible serves dynamically generated assessments based on educational video content, evaluates student performance, and issues secure, verifiable certificates.

---

## 🌟 Key Features

- **AI-Powered Assessments**: Ingests custom quiz payloads generated by an AI video analysis agent.
- **Dynamic Quiz Engine**: Supports multiple question types, including single-correct, multi-select, scenario-based, and code-output questions.
- **Progress Persistence**: Automatically caches the active quiz state in local storage, allowing candidates to safely refresh or resume their assessment.
- **Verifiable Credentials**: Generates professional, minimalist certificates with a unique ID format (`CERT-YYYY-XXXXXX`).
- **Secure ID Verification**: Includes an authentication portal to lookup and verify the validity of any issued certificate.
- **Attempt History**: Locally logs past assessment history, scores, and complete question reviews with correct/incorrect answers and explanations.

---

## 🛠️ Architecture & Technology Stack

Credible is built as a lightweight, high-performance web application:

- **Frontend**: Single Page Application (SPA) built using **React 19**, **React Router 7**, and vanilla **CSS3** with custom design tokens for a premium minimalist interface.
- **Utilities**: Client-side certificate image capture via **html2canvas** and PDF compiling via **jsPDF**.
- **Backend**: **Express.js** server hosting webhook ingestion endpoints and static asset routes.
- **Data Pipeline**: Integrates with external webhook streams (e.g. n8n workflows) that POST curriculum quiz payloads.

## 🚀 Quick Start

### 1. Installation
Install project dependencies for both the frontend and backend:
```bash
npm install
```

### 2. Development Mode
Launch both the Express backend and the Vite development server. Vite will run on port `5173` and automatically proxy API/webhook calls to the Express server on port `3000`.
```bash
# Start the Express server
npm run server

# Start the Vite development server (in a separate terminal)
npm run dev
```

### 3. Production Build & Execution
Build the optimized static frontend assets and launch the production server:
```bash
# Build the production bundle (creates the dist/ folder)
npm run build

# Start the server (serves the frontend on port 3000)
npm run start
```

## 📡 Webhook Integration

External workflows (such as n8n) can push new quiz definitions to the following endpoint:

- **Webhook URL**: `POST http://localhost:3000/webhook/quiz`
- **Headers**: `Content-Type: application/json`
- **Body**: Accepts a wrapped `{ quiz: { quiz_metadata, questions } }` or raw `{ questions }` payload structure.
